My website

Although frontend programming isn't my primary focus in my career, I decided to make this website my first publication because it was an interesting project that gave me a foundation in several languages I hadn’t worked with before. There are several reasons why I created this website, which I’ll go over below:

  • Create a "library": A place where I can document my knowledge and share it with others who may be looking for solutions or content related to programming.

  • Practice my learning: By executing projects from start to finish, I ensure I fully understand why they work and how the components fit together.

  • A dedicated space for theoretical content: A page where I can regularly review concepts and draw conclusions from my studies.

  • Showcase my skills: This website is a way to demonstrate my knowledge to recruiters and open doors for career opportunities in programming.

Despite its simplicity, this website was built entirely by me, and the process was very enjoyable. Here are some of the lessons I learned:

  • Folder organization: Initially, the site had only a few .html files, which were easy to access and edit. As the project grew, I had to rethink the folder structure and file organization to keep things manageable.

  • More than just programming: When I first started, I had no clear idea of the layout—I just wanted to code. After reworking the site a few times and getting stuck on the menu design, I realized how important planning and prototyping are before diving into coding.

  • HTML: I gained a good understanding of how to structure the layout using basic HTML commands effectively. It highlighted the importance of mastering even the most fundamental elements.

  • CSS: This was the one language I hadn’t had much exposure to before. As the project grew, I realized how essential consistent formatting is, especially as my CSS file started to grow too.

  • JavaScript: In my first experience using JavaScript for web development, I saw firsthand how powerful this language is. For this project, I used it to reuse the header and footer across all pages, allowing me to make changes in a single file and have them reflected site-wide.

  • Hosting: To put this website online, I purchased a domain and followed all the procedures to link it to the hosting platform, which in this case was Hostinger. Through this process, I gained a better understanding of how domain configurations and management work. Additionally, I took the opportunity to host my database online, allowing me to improve my CRUD applications.

Finally, this is a very interesting and engaging project that I will continue to improve whenever a new idea comes to mind.
